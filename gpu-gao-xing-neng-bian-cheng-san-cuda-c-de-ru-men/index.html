GPU高性能编程（三）CUDA-C的入门

GPU CUDA
GPU编程
发布日期:   2020-05-05
文章字数:   1.1k

本章将编写第一段CUDA C的代码，以及了解主机（Host）和设备（Device）之间编写代码的区别，并且查询CUDA的设备信息

Hello World

首先给出一个hello world的示例，这是编写在主机上的hello world程序。

#include <iostream>
int main(void){
    printf("Hello World!!");
    //或者使用std::cout<<"Hello World";
    return 0;
}

CPU以及系统的内存称之为主机(Host)，GPU及其内存称为设备(Device) 。
接下来编写在GPU上执行的代码，也就是在设备上执行代码。在GPU设备上执行的函数通常称为核函数(Kernel) 。

#include <iostream>
__global__ void kernel(void){}
int main(void){
  kernel<<<1,1>>>();
  std::cout<<"Hello World!";
    return 0;
}

其中：

  • 使用修饰符__global__，这个将告诉编译器，函数应该在设备上运行。函数kernel()将被交给编译设备代码的编译器，main()函数将被交给主机编译器
  • 采用修饰字符 <<<1,1>>>,第一个参数表示设备在执行核函数时使用并行线程块的数量，第二个参数表示每个线程块启动多少线程。

传递参数

对之前的hello world程序进行修改，使核函数不是空函数。

#include <iostream>

__global__ void add(int a,int b,int *c){
  *c=a+b;
}

int main(void){
  int c;
  int *dev_c;//在设备上的变量，会单独开辟内存
  cudaMalloc((void**)&dev_c,sizeof(int));//在设备上开辟内存
  add<<<1,1>>>(2,7,dev_c);
  cudaMemcpy(&c,dev_c,sizeof(int),cudaMemcpyDeviceToHost);//将设备上的dev_c传到主机c上，                                                                                                    cudaMemcpyDeviceToHost关键字表示从设备到主机。
  std::cout<<"2+7="<<c;
  cudaFree(dev_c);//记住一定要释放内存
}

关于cudaMalloc，cudaFree，cudaMemcpy的用法都写在注释中。

查询设备

cuda中有专门的函数对设备上的信息进行查询，cudaGetDeviceProperties函数会返回一个cuda的设备信息结构cudaDeviceProp。

struct cudaDeviceProp{
  char name[256];                  //设备的ASCLL字符串，名字
  size_t totalGlobalMem;                //设备上全局内存的总量，单位为字节
  size_t shareMemPerBlock;            //在一个线程块（Block）中可使用的最大共享内存，单位为字节
  int regsPerBlock;                            //每个线程块中可用的32位寄存器数量
  int warpSize;                                    //在一个线程束（Warp）中包含的线程数量
  size_t memPitch;                            //在内存复制中最大的修正量（Pitch），单位为字节
  int maxThreadsPerBlock;                //在一个线程块中可以包含的最大线程数量
  int maxThreadsDim[3];                    //在多维线程块数组中，每一维可以包含的最大线程数量
  int maxGridSize[3];                        //在一个线程格（Grid）中，每一维可以包含的线程块数量
  size_t totalConstMem;                    //常量内存的总量
  int major;                                        //设备计算功能集的主版本号
  int minor;                                        //设备计算功能集的次版本号
  int clockRate;                                
  size_t textureAlignment;            //设备的纹理对齐要求
  int deviceOverlap;                        //一个bool值，表示设备是否可以同时执行cudaMemory和核函数调用
  int multiProcessorCount;            //设备上多处理器的数量
  int kernelExecTimeoutEnabled;    //一个bool值，表示核函数是否存在运行时限制
  int integrated;                                //一个bool值，表示设备是否是一个集成GPU（属于芯片组的一部分）
  int canMapHostMemory;                    //一个bool值，表示设备是否可以将主机内存映射到CUDA设备地址空间
  int computeMode;                            //设备的计算模式：默认（Default）、独占（Exclusive）、禁止（Prohibited）
  int maxTexture1D;                            //一维纹理的最大大小
  int maxTexture2D[2];                    //二维纹理的最大维数
  int maxTexture3D[3];                    //三维纹理的最大维数
  int maxTexture2DArray[3];            //二维纹理数组的最大维数
  int concurrentKernels;                //一个bool值，表示设备是否支持在同一个上下文中同时执行多个核函数
}

简单获取一下设备信息：

#include <iostream>

int main(void){
  cudaDeviceProp prop;//构建一个结构, 用来获取设备信息

  int count;
  cudaGetDeviceCount(&count);//获取设备数量，可能有多块GPU
  for (int i=0;i<count;i++){
    cudaGetDeviceProperties(&prop,i);//获取对应设备上的信息
    std::cout<<"id "<<i<<" name: "<<prop.name;
  }
}

本章到此结束，其实CUDA C很类似于C。可以说是为GPU专门设计的C语言。
